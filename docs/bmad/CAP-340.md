### CAP-340 — Calibration & Observability

Goal: Maintain well-calibrated heads with cohorting, periodic refresh, and dashboards that track ECE, MAPE, and CI width against A++ gates.

Scope
- Heads: `share_prob`, `regret_prob` (probabilities) and `watch_time` (regression)
- Cohorts: platform, niche (category), and traffic band (low/med/high)
- Methods: Isotonic and Platt (select per cohort/head)
- Refresh cadence: weekly (manual trigger available via API)

APIs
- `GET /api/calibration/metrics?cohort=<string>` → per-head metrics `{ ece, ci_width_p90, updated_at, version }`
- `POST /api/calibration/refresh` → recomputes and persists calibrations for all cohorts; returns `{ ok, version, updated_at }`

Storage & Fallback
- Primary: Supabase tables `calibration_config(head, cohort, method, params, version, updated_at)` and `calibration_metrics(head, cohort, ece, ci_width_p90, mape?, updated_at)`
- Fallback: in-process memory for local/dev (no external dependency), seeded with safe defaults

Integration
- CAP-320 ranker consumes `calibrate(head, value, cohort)` to produce calibrated probabilities and CI widths
- `calibration_version` surfaced from calibrator

Observability
- Counters: calibration_refreshes, cohorts_calibrated, heads_calibrated
- Metrics exposed via API; simple admin page to visualize and verify

Acceptance
- ECE ≤ 0.03 across active cohorts
- 90% CI width ≤ 0.20
- `alg_version`/`calibration_version` present on responses
