# ALG-101 — Algorithm/Model Contract (A++)

Version: 1.0
Owner: ML Lead
Flag: `FF-AlgoAplusplus`

## Purpose
- Define inputs/outputs, calibration, exploration, and guardrails for the A++ multi‑objective recommender.

## Inputs
- User features: profile embeddings, recency, saturation, device, locale.
- Item features: content priors (CAP‑310), creator priors, freshness, entropy, policy risk.
- Context: session state, time‑of‑day, placement.

## Outputs
- Scores: `watch_time`, `share_prob`, `regret_prob` (all calibrated).
- Uncertainty: per‑head prediction intervals and overall uncertainty.
- Rank meta: `reason_codes` (top contributing features), `eligibility_flags`.

## Quality Gates
- Calibration per cohort: ECE ≤ 0.03; miscoverage on 90% CI in [0.85, 0.95].
- Early prediction MAPE ≤ 10% at 1,000 impressions.
- Regret floor: `regret_prob` above threshold enforces exposure cap.

### Thresholds (initial)
- Regret cap: if `regret_prob >= 0.18` and `watch_time_ci.upper < cohort_median`, apply exposure cap ≤ 20 impressions/user/day.
- Promotion: promote when `watch_time_ci.lower > cohort_median * 1.05` for 500 impressions.
- Demotion: demote when `watch_time_ci.upper < cohort_median * 0.95` for 300 impressions.

## Determinism & Versioning
- `alg_version` and `calibration_version` on every response.
- Stochastic exploration marked with `explore_id`; deterministic replay via seed.

## Cost & Latency
- p95 inference ≤ 120ms; feature fetch ≤ 40ms.
- Backpressure and max QPS defined in serving runbook.

## Observation & Events
- Emit: `EVT.Rec.ScoreServed`, `EVT.Rec.ItemPromoted`, `EVT.Rec.ItemDemoted`, `EVT.Rec.RegretCapped` (include `auditId`).

### Event payload (minimal)
```json
{
  "auditId": "uuid",
  "alg_version": "string",
  "calibration_version": "string",
  "user_id": "hashed",
  "item_id": "string",
  "scores": {"watch_time": 0.0, "share_prob": 0.0, "regret_prob": 0.0},
  "ci": {"watch_time": [0.0, 0.0]},
  "cohort_id": "device:ios|country:US|cat:education",
  "action": "served|promoted|demoted|regret_capped"
}
```

## Calibration
- Cohorted isotonic/Platt per device/country/category weekly; auto‑refresh; offline↔online parity checks.

## Explore/Exploit
- Top‑N re‑rank uses UCB/Thompson on uncertainty; promotion/demotion via calibrated CI thresholds.

## Constraints (Stage‑2 Re‑rank)
- Diversity, freshness minima, creator fairness, regret cap, novelty decay constraints.

## Rollout & Rollback
- Canary with switchback; stop‑loss: regret +15%, ECE > 0.05, p95 > 150ms.

## Change Control
- ADR required for head definitions, loss changes, or constraint weights.
