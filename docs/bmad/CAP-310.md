# CAP-310 — Cold‑Start Priors Service

Owner: Backend Lead  
Flag: `FF-AlgoAplusplus`

## Purpose
- Produce content‑only priors for new items before engagement data accrues. Reduce impressions‑to‑confidence and stabilize early ranking.

## Inputs
- `FeatureExtractor` outputs (visual/audio/text/structural/content).
- Creator historical priors (if creator known).
- Platform/placement context.

## Outputs
- `prior_watch_time`, `prior_share_prob`, `prior_regret_prob` with calibrated intervals.
- `prior_confidence` and `features_used_version`.

## API (HTTP)
- `POST /api/priors/item`
  - body: `{ features: ExtractedFeatures, creator_id?: string, platform?: string }`
  - resp: `{ prior_watch_time: number, prior_share_prob: number, prior_regret_prob: number, ci: { watch_time:[number,number] }, version: string }`

## Quality & Calibration
- Trained offline on historical content-only data; weekly recalibration per cohort.

## SLOs
- p95 ≤ 80ms; availability ≥ 99.9%.

## Events
- `EVT.Priors.Computed` with `auditId`, `version`, `cohort_id`.

## Rollout
- Canary behind `algo_aplusplus` flag; fallback to conservative defaults on error.
