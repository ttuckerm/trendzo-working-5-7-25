### CAP-320 — Two-stage ranker (multi-head + constrained re-rank)

Objective: Serve a ranked list that maximizes multi-objective utility with calibrated heads and safety constraints, enabling predictable improvements per A++ scorecard.

Stage-1 calibrated heads
- watch_time (seconds): expected watch time using content/structure features
- share_prob (0–1): probability of share given impression
- regret_prob (0–1): probability of post-click dissatisfaction/regret

Calibration
- Method: cohort-aware Platt/Isotonic hybrids; expose `calibration_version`
- Targets per ALG-101: ECE ≤0.03; 90% CI width ≤20% @1k imps
- Weekly refresh; cohorting by platform, niche, and traffic band

Stage-2 constrained re-rank (soft constraints with penalties and caps)
- Diversity: penalize near-duplicate topic/creator items; target ≥4 unique creators in top-10
- Freshness: boost recent uploads; monotone decay after 72h
- Regret floor: demote/cap items where `regret_prob > τ_regret` (emit RegretCapped)
- Novelty decay: downweight overexposed items (exposure caps carried to CAP-350)
- Fairness: per-creator exposure cap in top-K (K configurable; default 2 in top-10)

Composite objective (reference):
U = w1·norm_watch_time + w2·share_prob − w3·regret_prob − penalties(constraints)
Default weights: w1=0.6, w2=0.5, w3=0.8 (tunable via config)

API
- Route: `POST /api/recs/serve` (GET returns status)
- Flag gate: `algo_aplusplus` (403 when disabled)
- Request: { candidates?, topK?, platform?, cohort?, demo? }
- Response: { auditId, alg_version, calibration_version, topK, items[] }
  - items[i]: { id, rank, scores: { watch_time_s, share_prob, regret_prob, composite }, ci: { share_prob: {lo,hi,width}, regret_prob: {lo,hi,width} }, reasons[], constraintsApplied }

Events (ALG-101)
- EVT.Rec.ScoreServed: once per request with summary
- EVT.Rec.RegretCapped: per item capped by regret floor
  (ItemPromoted/Demoted emitted in CAP-330 exploration)

SLOs & acceptance gates (tie to a-plus-plus-scorecard.md)
- Predictability: MAPE ≤10% @1k imps; ECE ≤0.03; CI width ≤20%
- Reliability: p95 scoring ≤120ms; error rate ≤0.3%

Versioning
- `alg_version`: cap320-v0.1 (initial); increment on objective/constraint changes
- `calibration_version`: cal-v0.1 (placeholder until CAP-340 pipeline lands)

Observability
- Log inputs/outputs with `auditId` (hash-only where sensitive)
- Counters: served, regret_capped, constraint_hits (diversity, freshness, fairness)

Rollout
- Behind `algo_aplusplus`; canary → switchback per scorecard stop-loss rules
