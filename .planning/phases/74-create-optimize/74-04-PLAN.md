  <name>Task 5: Create OptimizePhase container</name>
  <files>src/components/workflow/phases/OptimizePhase.tsx</files>
  <action>
Create main Optimize phase container:

**Props interface:**
```typescript
interface OptimizePhaseProps {
  workflow: WorkflowRunWithDetails;
  onSave: (data: OptimizePhaseData) => Promise<void>;
  onBack: () => void;
  onContinue: () => void;
}

interface OptimizePhaseData {
  prediction_run_id: string | null;
  edited_content: {
    hook?: string;
    proof?: string;
    value?: string;
    cta?: string;
  };
  gate_check_results: GateWarning[];
}
```

**Implementation:**

1. **On mount/enter phase:**
   - Call POST /api/workflows/:id/optimize to trigger prediction
   - Show loading state while prediction runs
   - When complete, display results

2. **Data extraction:**
   - DPS score from prediction result
   - Section scores from Pack 1 attribute_scores
   - Gate warnings from gate-checks endpoint
   - Recommendations from Pack 2

3. **Layout:**
   ```
   ┌─────────────────────────────────────┐
   │  DPSScoreCard (centered, prominent) │
   ├─────────────────────────────────────┤
   │  SectionScores (2x2 grid)           │
   ├─────────────────────────────────────┤
   │  GateAWarnings (if any)             │
   ├─────────────────────────────────────┤
   │  AIRecommendations (max 3)          │
   ├─────────────────────────────────────┤
   │  Actions: Re-analyze | Continue     │
   └─────────────────────────────────────┘
   ```

4. **Inline editing:**
   - When "Fix This" clicked on warning, show inline text editor
   - When "Apply" clicked on recommendation, populate field
   - Changes stored in edited_content state
   - "Re-analyze" button re-runs prediction with edited content

5. **Re-analyze flow:**
   - Save edited content to Create phase step (step 3)
   - Re-trigger POST /api/workflows/:id/optimize
   - Display new results

6. **Continue button:**
   - Shows even with warnings (soft gates)
   - Calls onContinue to advance to Publish phase
  </action>
