  <name>Task 1: Create useVideoRecorder hook</name>
  <files>src/lib/hooks/useVideoRecorder.ts</files>
  <action>
Create a React hook for managing video recording state:

```typescript
interface UseVideoRecorderReturn {
  // State
  isRecording: boolean;
  isPreviewing: boolean;
  recordedBlob: Blob | null;
  previewUrl: string | null;
  error: string | null;

  // Stream management
  stream: MediaStream | null;

  // Actions
  startWebcam: () => Promise<void>;
  startScreen: () => Promise<void>;
  stopStream: () => void;
  startRecording: () => void;
  stopRecording: () => void;
  clearRecording: () => void;
}
```

Implementation:
1. **startWebcam()** - Use navigator.mediaDevices.getUserMedia({ video: true, audio: true })
2. **startScreen()** - Use navigator.mediaDevices.getDisplayMedia({ video: true, audio: true })
3. **startRecording()** - Create MediaRecorder from stream, collect chunks
4. **stopRecording()** - Stop MediaRecorder, create Blob from chunks, generate preview URL
5. **stopStream()** - Stop all tracks on the stream
6. **clearRecording()** - Revoke preview URL, clear blob

Handle errors gracefully:
- Permission denied for camera/mic
- No media devices available
- Screen sharing cancelled by user

MediaRecorder options: { mimeType: 'video/webm;codecs=vp9' } with fallback to 'video/webm'

**UI Structure:**
1. **Tab bar** - Three tabs: Upload | Webcam | Screen
   - Style: bg-gray-800 tabs with active tab highlighted (cyan gradient)

2. **Upload tab content:**
   - Drag-and-drop zone with dashed border
   - "Drop video here or click to browse"
   - Accept: video/mp4, video/webm, video/quicktime

   - Show selected file name + size after selection

3. **Webcam tab content:**
   - Video preview element (live stream)

4. **Screen tab content:**
   - "Share Screen" button to initiate capture
   - Same record/stop/preview flow as webcam

**Common elements:**
