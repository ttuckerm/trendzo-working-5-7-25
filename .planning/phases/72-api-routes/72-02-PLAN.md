<task type="auto">
  <name>Task 1: Create step update endpoint</name>
  <files>src/app/api/workflows/[id]/steps/[phase]/route.ts</files>
  <action>
Create Next.js App Router nested dynamic route with:

**GET /api/workflows/:id/steps/:phase** - Get specific step
- Get authenticated user
- Parse workflow ID and phase number from URL
- Get workflow and verify ownership
- Find step by phase_number
- Return WorkflowRunStep object

**PUT /api/workflows/:id/steps/:phase** - Update step data
- Get authenticated user, verify ownership
- Validate body with UpdateStepInputSchema
- Validate phase number is valid (1-6)
- Enforce back-navigation rule: can only edit current phase or earlier
- Call `updateStep(stepId, updates)` from workflow-db
- Updates can include: input_data, output_data, status, gate_check_results
- Return updated step

**Phase data structure by phase number:**
1. Research: { niche, demographics, content_purpose, goals, exemplars }
2. Plan: { pillar, seo_keywords, four_by_four: { hook, proof, value, cta } }
3. Create: { title, description, script, video_file_ref }
4. Optimize: { prediction_run_id, gate_checks, applied_suggestions }
5. Publish: { platform, scheduled_for, draft_id }
6. Engage: { published_url, performance_metrics, learnings }

Store these in input_data (user input) and output_data (computed results).
  </action>

<task type="auto">
  <name>Task 2: Create artifact management endpoints</name>
  <files>src/app/api/workflows/[id]/artifacts/route.ts</files>
  <action>
Create Next.js App Router route with:

**GET /api/workflows/:id/artifacts** - List all artifacts
- Get authenticated user, verify ownership
- Call `getWorkflowArtifacts(workflowId)` from workflow-db
- Support query param: `?type=research_summary|content_plan|video_script|prediction_result|publish_data|engagement_data`
- Return array of WorkflowRunArtifact objects

**POST /api/workflows/:id/artifacts** - Save new artifact
- Get authenticated user, verify ownership
- Validate body with SaveArtifactInputSchema
- Call `saveArtifact(workflowId, input)` from workflow-db
- Returns artifact with version number (auto-incremented for same type)
- Return 201 with created artifact

**GET /api/workflows/:id/artifacts/latest?type=X** - Get latest artifact of type
- Get authenticated user, verify ownership
- Parse type from query param (required)
- Call `getLatestArtifact(workflowId, type)` from workflow-db
- Return single artifact or 404 if none exists

Artifact versioning enables the edit history requirement:
"Artifacts may be overwritten when phase is re-edited"
(We keep old versions but getLatestArtifact returns newest)
  </action>
  <verify>TypeScript compiles without errors</verify>
  <done>Artifact CRUD working with versioning</done>
