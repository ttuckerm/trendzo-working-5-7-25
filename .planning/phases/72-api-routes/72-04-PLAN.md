Create WorkflowService class that wraps workflow-db with business logic:

```typescript
import * as workflowDb from './workflow-db';
import { runPredictionPipeline } from '@/lib/prediction/runPredictionPipeline';

export class WorkflowService {
  // Workflow lifecycle
  async startWorkflow(userId: string, metadata?: WorkflowMetadata);
  async completeWorkflow(workflowId: string);
  async abandonWorkflow(workflowId: string);

  // Phase transitions with validation
  async completePhase(workflowId: string, phaseNumber: number, outputData: StepOutputData);
  async validatePhaseCompletion(workflowId: string, phaseNumber: number): Promise<{ valid: boolean, errors: string[] }>;

  // Gate A checks for Optimize phase (soft gates)
  async runGateChecks(workflowId: string): Promise<GateCheckResult>;

  // Prediction integration for Optimize phase
  async runOptimizePrediction(workflowId: string): Promise<PredictionResult>;

  // Performance analysis for Engage phase
  async analyzePerformance(workflowId: string): Promise<PerformanceAnalysis>;
}
```

**Key business logic:**
1. `validatePhaseCompletion()` - Check required fields are filled before phase can complete
2. `runGateChecks()` - Execute Gate A checks (Hook effectiveness, Proof quality, CTA alignment)
3. `runOptimizePrediction()` - Trigger prediction pipeline and save results as artifact
4. `analyzePerformance()` - Compare predicted vs actual metrics (for Engage phase)

Gate checks should be SOFT - return warnings but allow proceeding (per CONTEXT.md decision).

Create Next.js App Router route with:

**POST /api/workflows/:id/optimize** - Run prediction for Optimize phase
- Get authenticated user, verify workflow ownership
- Verify workflow is at phase 4 (Optimize) or has completed phase 3 (Create)
- Get Create phase data (script/transcript from step output_data)
- Call `WorkflowService.runOptimizePrediction(workflowId)`
- This internally calls `runPredictionPipeline()` with the workflow content
- Save prediction results as 'prediction_result' artifact
- Update Optimize step with prediction run_id in output_data
- Return prediction results including Pack 1/2/3/V data

**GET /api/workflows/:id/optimize** - Get latest prediction results
- Get authenticated user, verify workflow ownership
- Get latest 'prediction_result' artifact for this workflow
- Return prediction results or 404 if not yet run

**POST /api/workflows/:id/optimize/gate-checks** - Run Gate A checks
- Get authenticated user, verify workflow ownership
- Call `WorkflowService.runGateChecks(workflowId)`
- Returns { passed: boolean, checks: [{ name, passed, warning? }] }
- Store results in Optimize step's gate_check_results

Gate A Checks (from UI designs):
1. Hook Effectiveness - "First 3 seconds grab attention"
2. Proof Quality - "Evidence supports claims"
3. CTA Alignment - "Clear call-to-action at end"

These map to Pack 1 attributes and 4x4 Method fields from Plan phase.
