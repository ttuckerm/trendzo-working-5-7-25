<task type="auto">
  <name>Task 1: Create global exemplar library endpoints</name>
  <files>src/app/api/workflows/exemplars/route.ts</files>
  <action>
Create Next.js App Router route with:

**GET /api/workflows/exemplars** - List user's exemplar library
- Get authenticated user
- Call `getExemplarLibrary(userId)` from workflow-db
- Support query params: `?platform=tiktok|youtube|instagram`, `?tags=tag1,tag2`
- Return array of ExemplarLibraryItem objects

**POST /api/workflows/exemplars** - Add exemplar to library
- Get authenticated user
- Validate body with AddExemplarInputSchema:
  {
    platform: 'tiktok' | 'youtube' | 'instagram',
    account_handle?: string,
    account_name?: string,
    video_url?: string,
    video_title?: string,
    metrics?: { views, likes, engagement },
    notes?: string,
    tags?: string[]
  }
- Call `addExemplar(userId, input)` from workflow-db
- Return 201 with created exemplar

**DELETE /api/workflows/exemplars/:id** - Remove from library
- Get authenticated user
- Verify user owns the exemplar
- Delete from exemplar_library table
- Return 204 No Content

This creates the global exemplar library that persists across workflows.

<task type="auto">
  <name>Task 2: Create workflow exemplar linking endpoints</name>
  <files>src/app/api/workflows/[id]/exemplars/route.ts</files>
  <action>
Create Next.js App Router route with:

**GET /api/workflows/:id/exemplars** - List exemplars linked to workflow
- Get authenticated user, verify workflow ownership
- Call `getWorkflowExemplars(workflowId)` from workflow-db
- Returns array of ExemplarLibraryItem objects (joined with exemplar_library)

**POST /api/workflows/:id/exemplars** - Link exemplar to workflow
- Get authenticated user, verify workflow ownership
- Body: { exemplar_id: string }
- Verify exemplar exists and belongs to user
- Call `linkExemplarToWorkflow(workflowId, exemplarId)` from workflow-db
- Return 201 with the linked exemplar

**DELETE /api/workflows/:id/exemplars/:exemplarId** - Unlink exemplar
- Get authenticated user, verify workflow ownership
- Delete from workflow_exemplars junction table
- Return 204 No Content

**POST /api/workflows/:id/exemplars/add-and-link** - Add new exemplar AND link to workflow
- Convenience endpoint: combines add to library + link to workflow
- Get authenticated user, verify workflow ownership
- Validate AddExemplarInput body
- Call `addExemplar(userId, input)` then `linkExemplarToWorkflow(workflowId, exemplarId)`
- Return 201 with created exemplar

This enables the "Both" decision:
"save exemplars to a global library AND link to current workflow"
