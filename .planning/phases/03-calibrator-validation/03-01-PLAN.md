**Phase:** 03-calibrator-validation
**Plan:** 01
**Status:** Ready

## Objective

Validate that the prediction calibrator rules work correctly in production environment with real prediction data.

## Context

The calibrator (`src/lib/prediction/prediction-calibrator.ts`) applies three rules:
1. **Rule 1**: Confidence penalty (0.7x) when no speech detected
2. **Rule 2**: DPS cap for silent videos (55 standard, 65 visual-first)
3. **Rule 3**: Log Pack V training features for model retraining

Guardrails:
- **Guardrail 1**: Gate DPS cap behind "no language signal" detection (transcript < 10 chars)
- **Guardrail 2**: Style-aware visual-first classification (style priority over niche)

### Task 3: Manual Smoke Test - Silent Video Scenario
**Type:** checkpoint
**Description:** User tests calibrator with silent video in /admin/upload-test

**Steps:**
1. Go to http://localhost:3000/admin/upload-test
2. Enter transcript: "" (empty) or very short (< 10 chars)
3. Select niche: "finance" (non-visual-first)
4. Submit prediction

**Expected:**
- If Pack V score < 50: DPS should be capped toward 55
- Confidence should be reduced by 0.7x
- Console logs show "[Calibrator] Rule 1 APPLIED" and/or "[Calibrator] Rule 2 APPLIED"

### Task 4: Manual Smoke Test - Visual-First Scenario
**Type:** checkpoint
**Description:** User tests calibrator with visual-first style

**Steps:**
1. Go to http://localhost:3000/admin/upload-test
2. Enter transcript: "" (empty)
3. Select niche: "satisfying" (visual-first niche fallback)
4. Submit prediction

**Expected:**
- DPS should use looser cap of 65 (not 55)
- Console shows "looser cap" or "visual-first" in adjustment reason

### Task 5: Verify Calibration Data in Database
**Type:** verification
**SQL Query:**
```sql
SELECT
  id,
  predicted_dps_7d,
  transcription_source,
  transcription_skipped,
  resolved_transcript_length
FROM prediction_runs
WHERE transcription_skipped = true
ORDER BY created_at DESC
LIMIT 5;
```
**Success Criteria:** Query returns rows with transcription metadata populated
